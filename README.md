# SECTION 1: FOUNDATIONS

## 1.1 WHAT IS MERMAID

Mermaid is a JavaScript-based diagramming and charting tool that uses text descriptions to generate diagrams dynamically. It was created by Knut Sveidqvist in 2014 and has since become the industry standard for documentation-as-code diagramming.

**Core Philosophy:**
- Diagrams should be as easy to maintain as code
- Text-based diagrams enable version control
- Documentation and diagrams should live together
- No vendor lock-in or proprietary formats

**Technical Foundation:**
Mermaid parses markdown-style syntax and renders SVG (Scalable Vector Graphics) elements in real-time. The output is resolution-independent, accessible, and easily embeddable.

## 1.2 WHY USE MERMAID OVER TRADITIONAL TOOLS

| Feature | Mermaid | Traditional Tools (Visio, Lucidchart, Draw.io) |
|---------|---------|------------------------------------------------|
| Version Control | Full (text-based) | Limited (binary formats) |
| Collaboration | Git-based | Proprietary sharing |
| Cost | Free and open source | Often subscription-based |
| Integration | Native in GitHub, GitLab, etc. | Manual export/import |
| Learning Curve | Syntax-based | GUI-based |
| Automation | Scriptable | Manual only |
| Accessibility | Screen-reader friendly | Variable |
| File Size | Kilobytes | Megabytes |

## 1.3 HOW MERMAID WORKS

1. User writes Mermaid syntax in a code block marked `mermaid`
2. The Mermaid renderer parses the text into an Abstract Syntax Tree (AST)
3. The AST is converted into SVG elements
4. SVG is injected into the DOM or exported as an image
5. Diagrams are interactive and scalable

## 1.4 BASIC SYNTAX STRUCTURE

All Mermaid diagrams follow this pattern:

    ```mermaid
    [diagram type]
    [content]
    [styling]
    ```

**Diagram Types:**
- `graph` / `flowchart` - Flowcharts
- `sequenceDiagram` - Sequence diagrams
- `classDiagram` - Class diagrams
- `stateDiagram-v2` - State diagrams
- `erDiagram` - Entity relationship
- `gitGraph` - Git graphs
- `gantt` - Gantt charts
- `pie` - Pie charts
- `mindmap` - Mind maps
- `timeline` - Timelines
- `sankey-beta` - Sankey diagrams
- `xyChart` - XY charts
- `C4Context` - C4 architecture
- `requirementDiagram` - Requirements
- `block-beta` - Block diagrams
- `quadrantChart` - Quadrant charts
- `journey` - User journeys

## 1.5 THE CODE BLOCK WRAPPER

Every Mermaid diagram must be wrapped in triple backticks with the language identifier `mermaid`:

    ```mermaid
    graph TD
        A[Start] --> B[End]
    ```

Without the proper wrapper, the code will display as raw text rather than rendering.

## 1.6 FIRST DIAGRAM: HELLO WORLD

    ```mermaid
    graph LR
        Hello[Hello World] --> Mermaid[Mermaid Diagram]
        Mermaid --> Done[Complete]
    ```

This creates a simple three-node flowchart demonstrating the fundamental concept.

## 2.1 ONLINE EDITOR METHOD

**Mermaid Live Editor**
URL: https://mermaid.live

Steps:
1. Navigate to mermaid.live
2. Enter your diagram code in the left panel
3. See instant preview on the right
4. Use the toolbar to download (PNG, SVG, PDF) or copy the code
5. Click "Share" to generate a permalink

Features:
- Real-time rendering
- Syntax highlighting
- Theme selector
- Export options
- Auto-save
- Split-screen view

**VS Code Setup:**
1. Install Visual Studio Code
2. Install the extension "Markdown Preview Mermaid Support" (by bierner)
3. Create a file with `.md` extension
4. Write Mermaid code blocks
5. Press `Ctrl+Shift+V` to open preview

**Other Editors:**
- **Sublime Text:** MarkdownPreview package
- **IntelliJ/WebStorm:** Built-in Mermaid support
- **Atom:** markdown-preview-enhanced package
- **Vim/Neovim:** markdown-preview.nvim

- README files
- Wikis
- Issues
- Pull requests
- Comments

# SECTION 3: FLOWCHARTS (GRAPH)

## 3.1 GRAPH DIRECTION TYPES

Mermaid flowcharts use the `graph` keyword followed by a two-letter direction code.

**Complete Direction Reference:**

| Code | Direction | Visual Flow |
|------|-----------|-------------|
| `graph TD` | Top to Bottom | Top → Bottom |
| `graph TB` | Top to Bottom (alias) | Top → Bottom |
| `graph LR` | Left to Right | Left → Right |
| `graph RL` | Right to Left | Right → Left |
| `graph BT` | Bottom to Top | Bottom → Top |

**Example - Top to Bottom:**

    ```mermaid
    graph TD
        Start --> Process --> End
    ```

**Example - Left to Right:**

    ```mermaid
    graph LR
        Start --> Process --> End
    ```

## 3.2 NODE SHAPES COMPLETE REFERENCE

Mermaid supports 12 distinct node shapes, each with specific use cases.

**Shape Syntax Table:**

| Shape Name | Syntax | Example | Use Case |
|------------|--------|---------|----------|
| Rectangle (default) | `A[Text]` | `Start[Begin]` | Standard process |
| Rounded Rectangle | `A(Text)` | `Start(Begin)` | Start/End points |
| Stadium | `A([Text])` | `Start([Begin])` | Alternative start |
| Circle | `A((Text))` | `Start((Begin))` | Connector points |
| Diamond | `A{Text}` | `Start{Begin}` | Decision/Question |
| Hexagon | `A{{Text}}` | `Start{{Begin}}` | Preparation step |
| Trapezoid | `A[/Text/]` | `Start[/Input/]` | Input/Output |
| Trapezoid (reverse) | `A[\Text\]` | `Start[\Output\]` | Output/Input |
| Cylinder | `A[(Text)]` | `Start[(Database)]` | Data storage |
| Asymmetric | `A>Text]` | `Start>Comment]` | Note/Annotation |
| Parallelogram | `A[/Text\]` | `Start[/Data\]` | Input/Output |
| Parallelogram (reverse) | `A[\Text/]` | `Start[\Data/]` | Output/Input |

**Complete Shape Example:**

    ```mermaid
    graph TD
        A[Rectangle]
        B(Rounded)
        C([Stadium])
        D((Circle))
        E{Diamond}
        F{{Hexagon}}
        G[/Trapezoid/]
        H[\Reverse Trapezoid/]
        I[(Cylinder)]
        J>Asymmetric]
        K[/Parallelogram\]
        L[\Parallelogram/]
    ```

## 3.3 ARROW TYPES AND CONNECTIONS

**Arrow Syntax Complete Reference:**

| Arrow Type | Syntax | Visual | Meaning |
|------------|--------|--------|---------|
| Solid arrow | `-->` | → | Standard flow |
| Solid arrow with label | `-->|label|` | →label→ | Annotated flow |
| Solid line (no arrow) | `---` | — | Connection |
| Solid line with label | `---|label|` | —label— | Annotated connection |
| Thick arrow | `==>` | ⇒ | Important flow |
| Thick arrow with label | `==>|label|` | ⇒label⇒ | Important annotated |
| Thick line | `===` | ≡ | Strong connection |
| Dotted arrow | `-.->` | ⇢ | Optional/weak flow |
| Dotted arrow with label | `-.->|label|` | ⇢label⇢ | Optional annotated |
| Dotted line | `-.-` | ··· | Weak connection |
| Two-way arrow | `<-->` | ↔ | Bidirectional |
| Two-way with label | `<-->|label|` | ↔label↔ | Annotated bidirectional |

**Arrow Combinations:**

    ```mermaid
    graph LR
        A[Solid] --> B[Arrow]
        C[Thick] ==> D[Arrow]
        E[Dotted] -.-> F[Arrow]
        G[Two Way] <--> H[Bidirectional]
        I[Labeled] -->|Important| J[Arrow]
        K[No Arrow] --- L[Line]
    ```

## 3.4 SUBGRAPHS (GROUPING)

Subgraphs allow logical grouping of nodes with visual boundaries.

**Syntax:**
    subgraph [Title]
        [nodes]
    end

**Example:**

    ```mermaid
    graph TD
        subgraph Frontend
            A[Web App]
            B[Mobile App]
        end
        
        subgraph Backend
            C[API Gateway]
            D[Database]
        end
        
        A --> C
        B --> C
        C --> D
    ```

**Nested Subgraphs:**

    ```mermaid
    graph TD
        subgraph Cloud
            subgraph Compute
                A[EC2 Instance]
                B[Lambda Function]
            end
            subgraph Storage
                C[S3 Bucket]
                D[RDS Database]
            end
        end
        
        A --> C
        B --> D
    ```

## 3.5 MULTIPLE CONNECTIONS

Nodes can connect to multiple other nodes in a single line.

**One-to-Many:**

    ```mermaid
    graph TD
        A --> B
        A --> C
        A --> D
    ```

**Many-to-One:**

    ```mermaid
    graph TD
        A --> D
        B --> D
        C --> D
    ```

**Chained Connections:**

    ```mermaid
    graph LR
        A --> B --> C --> D
    ```

## 3.6 LONG TEXT AND LINE BREAKS

Use double quotes for text containing spaces, line breaks, or special characters.

**Line Breaks with `<br>`:**

    ```mermaid
    graph TD
        A["This is a long text
        that spans multiple lines
        using HTML line breaks<br>Second line<br>Third line"]
        
        B["Contains: spaces, commas, and special characters"]
        
        A --> B
    ```

## 3.7 COMMENTS IN FLOWCHARTS

Use `%%` for single-line comments. Comments are not rendered.

    ```mermaid
    graph TD
        %% This is a comment - not rendered
        A[Start] --> B[Process]
        %% Another comment explaining the next step
        B --> C[End]
    ```

# SECTION 4: SEQUENCE DIAGRAMS

## 4.1 PARTICIPANTS

Participants represent entities that interact in the sequence.

**Basic Participants:**

    ```mermaid
    sequenceDiagram
        participant User
        participant Server
        participant Database
        
        User->>Server: Request
        Server->>Database: Query
        Database-->>Server: Result
        Server-->>User: Response
    ```

**Participant Aliases:**

    ```mermaid
    sequenceDiagram
        participant U as User
        participant S as Server
        participant DB as Database
        
        U->>S: Request
        S->>DB: Query
        DB-->>S: Result
        S-->>U: Response
    ```

**Participant Order:**
Participants appear in order of declaration. Use `participant` to define custom order.

## 4.2 ARROW TY
